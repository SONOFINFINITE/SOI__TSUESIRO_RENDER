# Twitch Bot для Render.com

Twitch бот с автоматическими командами и модерацией, готовый к деплою на бесплатном тарифе Render.com с функцией самопинга.

## 🚀 Возможности

- ✅ Автоматические команды чата (!вк, !бусти, !дис, !тг, !сыс, !дедот, !дэви, !вечно, !почухать)
- ✅ Автоматическая модерация CAPS (предупреждения и баны)
- ✅ Веб-сервер для поддержания активности на Render.com
- ✅ Автоматический самопинг (каждые 10 минут)
- ✅ Health check endpoints

## 📋 Команды бота

| Команда | Описание |
|---------|----------|
| `!вк` | Ссылки на паблики ВКонтакте |
| `!бусти` | Ссылка на Boosty |
| `!дис` | Ссылка на Discord |
| `!тг` | Ссылка на Telegram |
| `!сыс` | Случайное "сысье" приветствие |
| `!дедот` | Случайное действие с Дедотом |
| `!дэви` | Накормить Дэви |
| `!вечно` | ASCII-арт |
| `!почухать` | Выбрать случайного участника чата |

## 🛠️ Установка локально

1. Клонируйте репозиторий или скачайте файлы

2. Установите зависимости:
```bash
npm install
```

3. Создайте файл `.env` на основе `.env.example`:
```bash
cp .env.example .env
```

4. Заполните `.env` своими данными:
- `TWITCH_BOT_USERNAME` - имя бота
- `TWITCH_BOT_OAUTH` - OAuth токен бота (получить на https://twitchapps.com/tmi/)
- `TWITCH_CHANNEL` - канал для подключения
- `TWITCH_CLIENT_ID` - Client ID приложения Twitch
- `TWITCH_ACCESS_TOKEN` - Access Token с правами модератора
- `BROADCASTER_ID` - ID стримера
- `MODERATOR_ID` - ID модератора (обычно ID бота)

5. Запустите бота:
```bash
npm start
```

Для разработки с автоперезагрузкой:
```bash
npm run dev
```

## ☁️ Деплой на Render.com

### 1. Подготовка

1. Создайте репозиторий на GitHub и загрузите туда этот проект
2. Зарегистрируйтесь на [Render.com](https://render.com)

### 2. Создание Web Service

1. В панели Render нажмите "New +" → "Web Service"
2. Подключите ваш GitHub репозиторий
3. Настройте параметры:
   - **Name**: `twitch-bot` (или любое другое имя)
   - **Environment**: `Node`
   - **Build Command**: `npm install`
   - **Start Command**: `npm start`
   - **Plan**: выберите `Free`

### 3. Настройка Environment Variables

В разделе "Environment Variables" добавьте:

```
TWITCH_BOT_USERNAME=tsuesiro_bot
TWITCH_BOT_OAUTH=oauth:ваш_oauth_токен
TWITCH_CHANNEL=I_CbIC_I
TWITCH_CLIENT_ID=ваш_client_id
TWITCH_ACCESS_TOKEN=ваш_access_token
BROADCASTER_ID=144394710
MODERATOR_ID=1046743105
PORT=10000
RENDER_URL=https://ваше-имя-приложения.onrender.com
```

⚠️ **Важно**: `RENDER_URL` заполните после создания сервиса - скопируйте URL, который даст Render.com

### 4. Деплой

1. Нажмите "Create Web Service"
2. Дождитесь завершения деплоя (первый деплой может занять несколько минут)
3. После успешного деплоя скопируйте URL вашего приложения
4. Добавьте этот URL в переменную окружения `RENDER_URL` (Settings → Environment Variables)
5. Сохраните изменения - Render автоматически передеплоит приложение

### 5. Проверка работы

Откройте в браузере:
- `https://ваше-имя-приложения.onrender.com/` - главная страница со статусом
- `https://ваше-имя-приложения.onrender.com/health` - health check
- `https://ваше-имя-приложения.onrender.com/stats` - статистика бота

## 🔄 Самопинг

Бот автоматически пингует сам себя каждые 10 минут, чтобы не засыпать на бесплатном тарифе Render.com (который отключает неактивные приложения через 15 минут).

## 📝 Логи

Просмотр логов на Render.com:
1. Откройте ваш сервис в панели Render
2. Перейдите на вкладку "Logs"
3. Здесь вы увидите все логи бота в реальном времени

## 🔧 Получение токенов

### OAuth токен бота (TWITCH_BOT_OAUTH)
1. Перейдите на https://twitchapps.com/tmi/
2. Авторизуйтесь через аккаунт бота
3. Скопируйте полученный токен (формат: `oauth:xxxxx`)

### Access Token с правами модератора (TWITCH_ACCESS_TOKEN)
1. Создайте приложение на https://dev.twitch.tv/console/apps
2. Получите токен с scope: `moderator:manage:chat_messages`, `moderator:read:chatters`, `moderator:manage:banned_users`
3. Можно использовать https://twitchtokengenerator.com/

### Client ID (TWITCH_CLIENT_ID)
1. Создайте приложение на https://dev.twitch.tv/console/apps
2. Скопируйте Client ID из настроек приложения

## ⚠️ Ограничения бесплатного тарифа Render.com

- Приложение засыпает после 15 минут неактивности (самопинг решает эту проблему)
- 750 часов работы в месяц (достаточно для круглосуточной работы)
- Может быть небольшая задержка при "пробуждении" (cold start ~30 секунд)

## 🆘 Поддержка

При возникновении проблем проверьте:
1. Логи на Render.com
2. Правильность всех токенов и ID
3. Что бот имеет права модератора на канале
4. Что переменная `RENDER_URL` настроена правильно
